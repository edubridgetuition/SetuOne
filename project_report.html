1. Core Security & Row-Level Security (RLS)

SetuOne uses multi-tenant partitioning secured by PostgreSQL Row Level Security (RLS). Every query automatically isolates rows according to the authenticated tenant context.

Policy Boundaries

  • All tables include a company_id column linked to public.companies(id).
  • Users can only query, insert, or modify data rows containing their specific user account profile's company_id.

2. Module Chart Connections & SQL Mappings

This section maps every dynamic dashboard reporting chart directly to its database source schema and query logic.

📊 Helpdesk Ticketing Doughnut Chart

Visualizes: Counts of Open, Assigned, In-Progress, and Resolved tickets.

Table: public.tickets

SELECT status, COUNT(*) 
FROM public.tickets 
WHERE company_id = auth.current_company_id()
GROUP BY status;

🖥️ Asset Status Bar Chart

Visualizes: Inventory asset status metrics (In Use, Available, Maintenance, Retired).

Table: public.assets

SELECT status, COUNT(*) 
FROM public.assets 
GROUP BY status;

💰 Monthly Purchase Orders Cost Graph

Visualizes: Total expense breakdown logs filtered month-wise.

Table: public.purchase_orders

SELECT DATE_TRUNC('month', created_at) as month, SUM(total_amount) 
FROM public.purchase_orders 
GROUP BY month;

📦 Water Bottles & Consumption Trends

Visualizes: Daily delivered water bottles against returnable empty jugs.

Table: public.inventory_transactions joined with public.inventory_items

  • Deliveries: Transaction type = IN where name = '20L Water Jug'.
  • Returns: Transaction type = OUT or logged return quantities.

📊 Dynamic Report Templates & Custom Columns
