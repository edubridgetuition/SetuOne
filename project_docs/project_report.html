3. Database Structure & ERD

SetuOne enforces strict relational constraints ensuring database integrity across branches and locations:

Table Responsibility Ledger

Table Name Primary Purpose Relationships / Foreign Keys
public.tenants Subscriber scope and account validation. Isolated domain mapping.
public.companies Sub-brands (Orion Parks, Greenfield School). Linked to tenants(id).
public.branches Physical branch coordinates and regional settings. Linked to companies(id).
public.locations Self-referencing tree of Floors, Rooms, and Zones. Linked to buildings(id) and parent_id.
public.assets Holds barcode, category, serial number, cost, and warranty dates. Linked to categories, brands, and models.
public.inventory_transactions Real-time audit trails of item entries (IN) and consumption (OUT). Linked to inventory_items and branches.

4. JavaScript APIs & Repositories

The application interacts with data layers via asynchronous transactional JavaScript wrappers:

  • assetRepository.js: Handles dynamic category matching, barcode validation, manual floor updates, and depreciation timelines.
  • inventoryRepository.js: Runs real-time inventory adjustments and logs transactions on stock consumption events.
  • attendanceRepository.js: Binds security logs and coordinates geo-fenced checks.
