SetuOne ERP - Comprehensive Phase 8 & 9 Document

A non-technical explanation, user guide, flowchart, and technical database query map of the Reports & Analytics engine and Automation pipeline.

📌 1. Introduction & Phase Summaries

📊 Phase 8: Dynamic Reports

Transforms static worksheets into a configurable matrix. Managers can choose exactly which fields to see, save search parameters as templates (e.g. Weekly Attendance Audit), drill down from top summary cards, and export logs to Excel/CSV.

🤖 Phase 9: Automation Engine

A background automation engine monitoring thresholds and schedules. Trigger-based notifications route alerts (Low stock warnings, PPM schedules alerts, warranty expiration reminders) automatically to target department recipient groups.

📖 3. Phase 8: Non-Technical User Manual

  • 1
    Click Top Summary Ribbons (Drill Down)

    Clicking cards like "Open Tickets" or "Today Visitors" immediately filters the report workspace to that module. You don't need to manually configure dates or categories.

  • 2
    Choose Columns Display (Dynamic Grid)

    Check/uncheck checkboxes under "Select Fields". Columns in the table below will instantly vanish or appear. This lets you construct custom report views (e.g. view tickets without priority details).

  • 3
    Save Search Template

    Once you are happy with your filters and selected columns, type a name in "Saved Reports Filters" box (e.g. `June Complaint Audit`) and click "Save Current". Next time, click the template to instantly load it.

  • 4
    Export Spreadsheet

    Click the green "Export CSV" button. Your browser will instantly download a spreadsheet file of the active table.

💻 4. Phase 8: Technical Report Database Maps

Here are the database connections used to feed the dynamic reports table:

Table: public.saved_report_filters

Stores user-specific customized search state parameters.

SELECT filter_name, report_type, filters_payload 
FROM public.saved_report_filters 
WHERE profile_id = auth.uid();
Table: public.report_templates

Holds company-wide configured reporting templates (e.g. Weekly Facility Roster).

SELECT template_name, config_payload 
FROM public.report_templates 
WHERE company_id = 'company-uuid-context';

📖 6. Phase 9: Non-Technical User Manual

  • 1
    Create Alert Templates

    Navigate to "Email Templates". Write custom alerts subjects and body text. Use tags like {{employee_name}} or {{item_name}} to let the system fill details automatically.

  • 2
    Group Target Stakeholders

    In "Recipient Groups", create a group like `Admin Officers` and insert target email addresses. You don't have to input individual emails every time you configure a rule.

  • 3
    Activate Rules Constraints

    In "Automation Rules", configure triggers (e.g. Trigger 'Low Stock Alert' when closing stock goes below reorder level). Map this to the templates and groups you created.

  • 4
    Monitor Dispatch Audit Logs

    Go to the "Automation Audit logs" tab to review system-triggered logs. You will see which notifications succeeded, which channel was selected, and latency delay stats.

💻 7. Phase 9: Technical Automation Schema Mappings

Table: public.notification_rules

Houses constraints, conditions, version counts, and maintenance pauses.

SELECT rule_name, trigger_type, condition_payload, is_paused 
FROM public.notification_rules 
WHERE is_active = true;
Table: public.notification_queue

Priority queue buffer storing dispatches.

SELECT recipient, subject, body_text, priority, status 
FROM public.notification_queue 
WHERE status = 'Pending' 
ORDER BY priority DESC, created_at ASC;

🚀 8. Steps for Managers to Verify Features

✅ Phase 8 Verification Steps:
  1. Log in as manager and select Reports tab.
  2. Click Open Tickets card at the top ribbon. Check if report filters update.
  3. Uncheck columns in selector checkboxes. Check if table layout updates instantly.
  4. Input a filter name in the right sidebar, click Save Current. Refresh page and click the saved filter card. Check if old configurations load back.
  5. Click Export CSV. Check if your browser triggers download.
✅ Phase 9 Verification Steps:
  1. Select User Roles ➡️ Automation Rules tab in sidebar.
  2. Go to Email Templates tab. Create a test template (e.g. `Water warning`).
  3. Go to Recipient Groups. Create a group (e.g. `Maintenance Heads`).
  4. Go to Automation Rules. Link your template, group, set condition values, and click Activate Rule. Check if rule shows in bottom table.
  5. Go to My Alerts Preferences. Try toggling checkmarks to verify settings update.
  6. Click blue Trigger Scheduler Check button on right panel. Go to Automation Audit logs tab. Check if simulated logs appear in list.
